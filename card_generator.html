CSVファイルの書き方

1行目にヘッダー（列名）を書き、2行目以降にカードのデータを1行ずつ記述します。

1行目: ヘッダー (この順番で記述してください) 
カード名,色,タイプ,背景,イラスト,効果説明,フレーバー,フレーバー名,キラ
2行目以降: カードデータの例 
一撃必殺,赤,,,"images/sword.png","このカードは敵を即座に倒す。","「これで終わりだ！」,英雄,false
伝説の秘宝,虹,FF,hologram_geometric.png,"items/treasure.png",ゲームに勝利する。,,,true
ただの石,白,,,,何の効果もない。,,,
各列の詳細説明
  • カード名: (必須) カードの名前。ルビ記法も使えます例: `超新星(スーパーノヴァ)`)
  • 色: (必須) ドロップダウンリストにある色名。例: `赤`, `青`, `虹`
  • タイプ: (任意) カードの見た目を変更します。空欄の場合は「標準」になります。
    • `FF`: 画像がカード全体に表示される「フルフレーム」
    • `CF`: カード名に白い縁取りが付く「文字枠なし」
    • `FFCF`: 上記の両方を適用
  • 背景: (任意) 画像が透過PNGの場合の背景。例: `◇`
    • `◇`: ホログラム（幾何学）
    • `☆`: ホログラム（粒子）
  • イラスト: (任意) カード画像のファイル名。手順2でアップロードしたフォルダ内にある必要があります。
  • 効果説明, フレーバー, フレーバー名: (任意) それぞれのテキスト。
  • キラ: (任意) キラカードにするかどうか。`true`を指定するとキラカード(APNG)になります。`false`または空欄の場合は通常のカード(PNG)になります。

JSONファイルの書き方

キーと値をセットにしたオブジェクトの配列として記述します。

[
  {
    "cardName": "伝説の秘宝",
    "color": "虹",
    "type": "FF",
    "background": "◇",
    "image": "items/treasure.png",
    "effect": "ゲームに勝利する。",
    "flavor": "",
    "speaker": "",
    "sparkle": true
  },
  {
    "cardName": "ただの石",
    "color": "白",
    "effect": "何の効果もない。",
    "sparkle": false
  }
]

※JSONでは、指定しないキーは自動的に空欄またはデフォルト値（`sparkle`は`false`）として扱われます。
